El Desafío

Crear una experiencia de bienestar personalizada para mujeres en transición hormonal que no se sintiera estática ni genérica. El sistema necesitaba adaptar recomendaciones médicas complejas basándose en el input diario variable (síntomas, ánimo, sueño) de la usuaria.

  • Gestión de estado complejo con múltiples variables de salud.
  • Integración de IA con baja latencia y alta relevancia contextual.

La Solución

Implementé una Arquitectura Híbrida. Un motor de reglas determinista asigna un "Plan Base" seguro y validado médicamente, mientras que una capa de IA Generativa (Gemini) procesa los registros diarios para personalizar el tono, la motivación y los micro-ajustes del plan.

  • Arquitectura MVVM limpia para separar lógica de UI.
  • Repositorio local para planes base (offline-first).
  • Prompt Engineering avanzado para respuestas JSON estructuradas.

Ingeniería de IA

Cómo integré Gemini para generar respuestas estructuradas (JSON) y dinámicas.

DashboardViewModel.kt 
private suspend fun generateDashboardContent(
    profile: UserProfile, 
    log: DailyLog?
): DashboardContent {
    
    // Prompt Engineering Contextual
    val prompt = """
        Eres 'Lunara', asistente experta.
        Usuaria: ${profile.nombre}. Objetivo: ${profile.objetivos.first()}.
        Datos de hoy: Energía ${log?.energyLevel}/5, Sueño ${log?.sleepQuality}/5.
        
        Genera un JSON con:
        1. "greeting": Saludo empático corto.
        2. "dailyPlan": Lista de 3 tareas breves (Nutrición, Ejercicio).
        NO uses emojis en el JSON.
    """.trimIndent()

    // Llamada Asíncrona a Gemini
    val response = generativeModel.generateContent(prompt)
    
    // Parseo seguro a Data Class de Kotlin
    return response.responseAs() 
           ?: DashboardContent.Error
}

Prompt Engineering Dinámico

El prompt no es estático. Se construye en tiempo de ejecución inyectando variables del UserProfile (Firestore) y el DailyLog (Tracking). Esto garantiza que la IA tenga contexto total antes de responder.

Respuesta Estructurada (JSON Mode)

En lugar de texto libre, forzamos a Gemini a devolver un JSON que mapea directamente a nuestras data classes de Kotlin (@Serializable). Esto permite renderizar UI nativa (Cards, Listas) a partir de la "imaginación" de la IA.

Gestión de Estado con Coroutines

Todo el proceso ocurre fuera del hilo principal usando viewModelScope y suspend functions, garantizando que la UI nunca se congele mientras la IA "piensa".
