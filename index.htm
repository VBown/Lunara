                <p class="text-slate-600 leading-relaxed mb-6">

                    Crear una experiencia de bienestar personalizada para mujeres en transición hormonal que no se sintiera estática ni genérica. El sistema necesitaba adaptar recomendaciones médicas complejas basándose en el <em>input</em> diario variable (síntomas, ánimo, sueño) de la usuaria.

                </p>

                    </li>

                    <li class="flex items-start gap-3 text-slate-600">

                        <i class="ph-bold ph-warning-circle text-red-400 mt-1"></i>

                        <span>Integración de IA con baja latencia y alta relevancia contextual.</span>

                    </li>

                </ul>

            </div>

            <div>

                <h2 class="text-3xl font-bold text-slate-800 mb-6 flex items-center gap-3">

                    <i class="ph-duotone ph-lightbulb text-green-500"></i> La Solución

                </h2>

                <p class="text-slate-600 leading-relaxed mb-6">

                    Implementé una <strong>Arquitectura Híbrida</strong>. Un motor de reglas determinista asigna un "Plan Base" seguro y validado médicamente, mientras que una capa de <strong>IA Generativa (Gemini)</strong> procesa los registros diarios para personalizar el tono, la motivación y los micro-ajustes del plan.

                </p>

                <ul class="space-y-3">

                    <li class="flex items-start gap-3 text-slate-600">

                    <li class="flex items-start gap-3 text-slate-600">

                    <div class="mb-8">

                        <h3 class="text-xl font-bold text-lunaraPrimary mb-2">Prompt Engineering Dinámico</h3>

                        <p class="text-slate-400 text-sm leading-relaxed">

                            El prompt no es estático. Se construye en tiempo de ejecución inyectando variables del <code>UserProfile</code> (Firestore) y el <code>DailyLog</code> (Tracking). Esto garantiza que la IA tenga contexto total antes de responder.

                        </p>

                    </div>

                    <div class="mb-8">

                        <h3 class="text-xl font-bold text-lunaraSecondary mb-2">Respuesta Estructurada (JSON Mode)</h3>

                        <p class="text-slate-400 text-sm leading-relaxed">

                            En lugar de texto libre, forzamos a Gemini a devolver un JSON que mapea directamente a nuestras data classes de Kotlin (<code>@Serializable</code>). Esto permite renderizar UI nativa (Cards, Listas) a partir de la "imaginación" de la IA.

                        </p>

                    </div>

                    <div>

                        <h3 class="text-xl font-bold text-blue-400 mb-2">Gestión de Estado con Coroutines</h3>

                        <p class="text-slate-400 text-sm leading-relaxed">

                            Todo el proceso ocurre fuera del hilo principal usando <code>viewModelScope</code> y <code>suspend functions</code>, garantizando que la UI nunca se congele mientras la IA "piensa".

                        </p>
